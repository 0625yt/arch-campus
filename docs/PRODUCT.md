## 0. Executive Summary

### 한 줄 정의
**강의자료와 강의계획서를 넣으면, 오늘 해야 할 공부·과제·발표·시험·자격증 준비를 자동으로 정리하는 한국 대학생 전용 AI 학기 운영 OS.**

### 핵심 차별점 5가지
1. **강의계획서 자동 일정화** — 한국 대학 양식 특화 (NotebookLM·Quizlet 절대 못 함)
2. **위저드 대화 학습 보조** — AI 못 쓰는 학생도 결과물 만드는 단계별 가이드
3. **학기 운영 + 시험·자격증 + 공모전 통합** — 한 앱으로 대학 4년 + 자격증 + 취업까지
4. **친구 초대 기반 학과·과목 단위 viral 성장** — 에브리타임 우회
5. **성적·합격 결과 추적** — 한국 EdTech 어디도 안 가진 데이터 자산

### MVP 범위 (12주, 8개 기능)
P0 핵심 5개 + P1 결정 동력 3개. 나머지는 Phase 2 이후.

### 로드맵 요약
| Phase | 기간 | 목표 |
|---|---|---|
| Phase 0 | 0~1개월 | 시장 검증 + 대기열 100명 |
| Phase 1 (MVP) | 2~4개월 | 8개 핵심 기능 출시 |
| Phase 2 | 5~9개월 | 자격증·시험 통합 + 결제 검증 |
| Phase 3 | 10~18개월 | 공모전·팀플·취업 확장 |
| Phase 4 | 19~24개월 | 자격증 전문 분기 + 글로벌 검토 |

### 천억 매출 경로
한국 대학생 (Year 1) → 자격증·평생학습 (Year 2~3) → 글로벌 비영어권 학습 시장 (Year 4~5)

## 1. 제품 정의

### 1-1. 비전
한국 대학생이 입학부터 졸업까지 **모든 학습·평가·진로 활동을 하나의 AI로 운영**할 수 있게 한다.

### 1-2. 타깃 사용자
| 1차 (Year 1) | 2차 (Year 2~) |
|---|---|
| 한국 일반대학 학부생 | 자격증 준비 직장인·취준생 |
| 컴공·공대 우선 | 일반 자격증·공무원 시험 |
| 시험기간·과제 마감에 압박받는 학생 | 평생학습자 |

### 1-3. 핵심 Job-To-Be-Done
1. **시험기간**: 노트 다시 읽는 비효율 제거 → 즉시 문제·복습으로 전환
2. **과제·발표**: AI를 어떻게 쓸지 막막함 → 원하는 대로 결과물 도출
3. **일정**: 강의계획서·과제 마감 놓침 → 자동 추출 + Today 대시보드
4. **자격증·시험**: 강의자료가 없는 시험 준비 → 사용자 자료 업로드 + 학습 루프
5. **취업·실습**: 공모전·대외활동 정보 흩어짐 → 한 화면 + 필터링

### 1-4. 우리가 이길 수 있는가 (Competitive Moat)
| Moat | 설명 |
|---|---|
| **한국 강의계획서 파싱 사전** | 대학별·학과별 양식 표준화 데이터, 글로벌 제품은 절대 못 만듦 |
| **한국 대학 학습 맥락 데이터** | 성적·합격 코호트 분석, 5년 누적 시 매각가 1조 평가 가능 (확인 필요) |
| **위저드 UX** | "AI 못 쓰는 80%"를 위한 진입장벽 제거 |
| **친구 초대 viral** | 학과·과목 단위 자연 확산, CAC 1/5 |
| **시장 트렌드 지속적 학습 효과** | NotebookLM·Quizlet의 약점 정확히 인식하고 있음 |

## 2. 최종 기능 명세 (12개 기능 통합)

### 2-1. P0 핵심 5개 (MVP 필수)

#### 기능 1. 학습 루프 (Study Engine)
**한 줄 정의**: 자료 업로드 → 요약 → 문제 생성 → 오답 저장 → 간격 반복 복습

**구체 사양**:
- 지원 포맷: PDF, HWPX, PPTX, DOCX, TXT, MD (HWP는 변환 안내)
- 요약 출력: 핵심 개념 표 + 3줄 요약 + 단원별 키워드
- 문제 생성: 객관식·주관식·서술형, 난이도 3단계
- 출처 근거 표시: 각 문제마다 "근거: 8페이지 3문단" 명시
- 오답노트: 자동 누적, 단원·개념 태깅
- 복습 큐: **FSRS 알고리즘** 기반 간격 반복 (SM-2 대비 효율 30% 우수, 확인 필요)
- 푸시 알림: 매일 복습 시간 알림

**검증 루프 (필수)**:
```
문제 생성 → 출처 근거 span 확인 → 정답 유일성 검사 →
보기 충돌 검사 → 통과 시 사용자 노출
```
이 검증이 없으면 "복수 정답"·"근거 없는 문제"가 생긴다. 신뢰도 즉시 붕괴.

#### 기능 2. 강의계획서 자동 일정화 (Syllabus Engine)
**한 줄 정의**: 강의계획서 PDF 업로드 → 시험·과제·발표 일정 자동 추출 → 캘린더 등록

**구체 사양**:
- 지원 포맷: PDF, HWPX (HWP는 변환 안내)
- 추출 항목: 시험일, 과제 마감, 발표 일정, 평가 비율, 주차별 계획
- 신뢰도 점수 표시: 80% 이상 자동 등록, 80% 미만 사용자 확인
- 캘린더 통합: Google Calendar·Apple Calendar 양방향 동기화
- D-day 알림: 7일·3일·1일 전 푸시
- 시간표 자동 등록: 강의계획서에서 강의 요일·시간 추출

**핵심 UX 원칙**: 100% 자동 환상 버려라. **"확인하기 쉬운 80% 자동"이 정답**.

#### 기능 3. Today 대시보드 (Home Screen)
**한 줄 정의**: 매일 앱을 열게 만드는 단 하나의 화면

**구체 사양**:
```
[Today - 5월 1일 금요일]

🔥 지금 해야 할 것 (5분 안쪽)
   - 운영체제 복습 12문제 [지금 시작]

⏰ 마감 임박
   - 자료구조 과제 D-2 (3시간 안쪽) [위저드 시작]
   - 데이터베이스 발표 D-5 [위저드 시작]

🎯 시험·자격증
   - 정보처리기사 모의고사 D-30
   
📌 추천
   - "이번 주 발표 준비" 위저드 [실행]
   - 신 공모전 3건 (전공 매칭) [보기]
```

**설계 원칙**:
- 앱 인터페이스 메인 노출 금지. 카드형 대시보드.
- 1초 안에 "오늘 무엇부터" 보임
- 모든 카드는 단일 액션 버튼 (지금 시작 / 위저드 시작 / 보기)

#### 기능 4. 친구 초대 Viral Loop
**한 줄 정의**: 같은 학과·과목 친구를 데려오면 둘 다 무료 Pro 1주

**구체 사양**:
- 초대 코드 생성 (학과 정보 자동 매칭)
- 같은 과목 듣는 친구 5명 모이면 "스터디 모드" 잠금 해제
- 스터디 모드: 익명 통계 공유 (우리 과목 평균 정답률 등). 자료 공유 X (저작권 차단)
- 학과별 가입 리더보드 (게이미피케이션)

**중요 차별**: 커뮤니티 기능 X. 1:1 또는 소그룹 학습 보조. 모더레이션 부담 없음. 에브리타임과 충돌 없음.

#### 기능 5. 성적·합격 추적 (Outcome Tracking)
**한 줄 정의**: 사용자의 실제 결과를 누적해서 데이터 자산으로 만든다

**구체 사양**:
- 학기 시작: "이번 학기 목표 학점" 입력 (옵션)
- 학기 말: "실제 받은 학점" 입력 → 보상 (1주 Pro 등)
- 자격증: "응시일·합격 여부" 입력
- 코호트별 평균 학점 향상 익명 통계 표시
- 개인화: 학점 낮으면 더 적극적 알림, 높으면 자격증·심화 추천

**전략적 의미**: 이게 **천억 회사의 진짜 해자**. 한국 대학생 학습 결과 데이터는 NotebookLM·Quizlet이 절대 못 모은다.

### 2-2. P1 결정 동력 4개 (MVP 1차 또는 2차 출시)

#### 기능 6. 위저드 + 프롬프트 라이브러리 (구 MD 템플릿 실행기)
**한 줄 정의**: AI 못 쓰는 학생도 단계별로 답하면 결과물이 나오는 "AI 가이드"

**핵심 인사이트**: 학생의 80%는 ChatGPT를 "글 써줘"로만 쓴다. 우리가 검증된 프롬프트와 단계별 가이드를 제공하면 진입장벽이 사라진다. 이게 **진짜 차별점**이다.

**구체 사양 — 두 가지 모드**:

**모드 A: 위저드 (초보자용, 기본값)**
```
[발표 준비 위저드]
1단계: 발표 주제는 무엇인가요? [입력]
2단계: 발표 시간은? [10분 선택]
3단계: 청중 수준은? [같은 과 학생들]
4단계: 교수님이 강조한 평가 기준은? [입력]
5단계: 참고 자료를 올려주세요 [업로드]
→ [발표 구조 + 슬라이드 초안 + 발표 대본 + 예상 질문] 자동 생성
```

**모드 B: 프롬프트 라이브러리 (고급 사용자용)**
```
[라이브러리]
- 발표자료_구조화
- 발표_QA_예상질문
- 리포트_초안작성
- 교수_요구사항_체크
- 기출형_문제생성
- 오답_원인분석
- 벼락치기_요약
- 팀플_역할분배
- 합의록_정리
- 자기소개서_초안
- 면접_예상질문
- 공모전_제안서_구조

→ 카테고리 선택 → 변수 입력 → 실행 → 결과 저장/수정
```

**전략**: 초보자는 위저드(80%)로, 고급 사용자는 라이브러리(20%)로. 같은 백엔드, 다른 진입점.

**제공할 위저드/템플릿 리스트 (출시 시 12종)**:

| 카테고리 | 위저드 이름 | 입력 변수 | 출력 |
|---|---|---|---|
| 발표 | 발표자료 구조화 | 주제·시간·청중·평가기준·자료 | 슬라이드+대본+예상질문 |
| 발표 | Q&A 예상질문 | 발표 구조·핵심 주장 | 질문 10개+답변 초안 |
| 과제 | 리포트 구조 설계 | 주제·분량·문체·필요요소 | 목차+섹션별 가이드 (※초안 X, 학생이 직접 작성하도록) |
| 과제 | 교수 요구사항 체크 | 과제 공지·평가 기준 | 가점 슈팅 체크리스트 |
| 시험 | 기출형 문제 생성 | 자료·난이도·문제수·형식 | 객관식/주관식/서술형 |
| 시험 | 오답 원인 분석 | 문제·내 답·정답·해설 | 오답 유형+보완 개념 |
| 시험 | 벼락치기 학습 계획 | 시험 범위·남은 시간 | 30분/1시간/3시간 학습안 |
| 팀플 | 역할 분배 | 주제·팀원수·마감 | 역할표+일정+체크포인트 |
| 팀플 | 합의록 정리 | 회의 텍스트/녹취 | 합의록+숙제아이템 |
| 진로 | 자기소개서 구조화 | 회사·직무·내 경험 | 문항별 구조 가이드 |
| 진로 | 면접 예상질문 | 회사·직무·이력서 | 질문 20개+답변 프레임 |
| 진로 | 공모전 제안서 구조 | 공모전·팀·아이디어 | 제안서 목차+핵심 메시지 |

**중요한 UX 결정**: "리포트 초안 작성"은 학칙·윤리 위험. 우리는 **초안을 써주지 않고 "구조 + 가이드 + 체크리스트"만 제공**. 이게 차별화이자 보호.

#### 기능 7. 시험 후 회고 (Post-Mortem)
**한 줄 정의**: 시험 끝난 다음 학기·다음 시험을 위한 학습 계획 자동 생성

**구체 사양**:
- 시험 후 입력: 잘 본 부분·못 본 부분·시간 부족 단원
- AI 분석: 오답 패턴 → 약점 단원 → 다음 학습 우선순위
- 다음 학기 자동 반영: 같은 교수·같은 학과 데이터로 학습 가이드
- 자격증 시험 회고도 동일 구조

**전략적 의미**: 학기 종료 후 이탈을 막는 **유일한 기능**. 학기 → 회고 → 다음 학기의 연속 루프 형성. retention의 핵심.

#### 기능 8. 공모전·대외활동 플레이어
**한 줄 정의**: 전공·관심사 기반 공모전·인턴·대외활동 추천

**구체 사양**:
- 데이터 소스: 위비티, 캠퍼스픽, 맞커리어 등 공개 데이터 + 자체 플레이어 (확인 필요: 크롤링 vs 제휴)
- 필터: 전공·학년·마감일·상금·온/오프라인
- 알림: 신 공모전 + 마감 임박
- 매칭: 사용자 학과·관심사 기반 자동 추천
- 위저드 연동: "공모전 제안서 구조" 위저드와 연결

**전략**: 추천 알고리즘 복잡하게 만들지 말고 **플레이어 + 필터링**부터. 사용자 1만 명 시점 개인화 강화.

**중요 의사결정**: 공모전 데이터 출처
- 옵션 A: 직접 플레이어 (월 2~3시간 운영 비용)
- 옵션 B: 위비티·캠퍼스픽 API 또는 제휴 (확인 필요)
- 옵션 C: 사용자 제보 + 자체 검색

→ MVP는 **옵션 A**로 시작. 사용자 1만 명 도달 후 옵션 B 검토.

### 2-3. P2 확장 기능 3개 (Phase 2~3)

#### 기능 9. 시험·자격증 통합 모듈

**확장 시점 결정 기준** (5개 중 3개 충족):

**구체 사양**:

- 자료 통합 (개인 자료의 일부만 팀 공유 옵션)
- 역할 분배 위저드
- 합의록 자동 정리
- 팀 발표 위저드

**개발 복잡도**: 매우 높음. 권한·공유·동기화 모두 어려움. **MVP 절대 제외**, 사용자 요구 누적 후 추가.

**한 줄 정의**: 자기소개서·면접·이력서 위저드 통합

**구체 사양**:

**전략적 의미**: 졸업 직전 사용자 ARPU 폭증. **취업 시즌 패스** 49,000원/3개월 같은 상품 가능.

## 3. 데이터·AI 아키텍처 (요약)

### 3-1. 핵심 파이프라인
```
파일 업로드 → 포맷 판별 → 텍스트 추출/OCR → 블록 정규화 →
청킹 → 임베딩 → 벡터 저장 → 검색/RAG →
요약/문제/일정/위저드 결과 생성 → 검증 → 사용자 노출
```

### 3-2. 기술 선택 (확인 필요, 최종 검토 필요)
| 영역 | 권장 |
|---|---|
| 디지털 PDF 파싱 | PyMuPDF + pdfplumber |
| 스캔 OCR | PaddleOCR (PP-StructureV3) 또는 업스테이지 Document Parse |
| HWPX 파싱 | python-hwpx |
| 임베딩 | OpenAI text-embedding-3-small ($0.02/1M tokens) |
| 생성 모델 (저비용) | GPT-4.1 mini ($0.40 입력 / $1.60 출력) |
| 생성 모델 (고품질) | Claude Sonnet 4.6 또는 GPT-4.1 (위저드 결과물) |
| 벡터 DB | Supabase pgvector ($25/월~) |
| 문서 저장 | Cloudflare R2 또는 S3 |
| 백엔드 | Next.js + Supabase 또는 FastAPI + PostgreSQL |
| 프론트엔드 | Next.js — **반응형 웹 1급 (Mobile · iPad · Desktop 동등 지원)**. React Native 네이티브 앱은 Phase 4+ 재검토 |

### 3-3. 모델 라우팅 전략
| 작업 | 모델 | 이유 |
|---|---|---|
| 문서요약 | GPT-4.1 mini | 비용 |
| 문제 생성 | GPT-4.1 mini | 비용 |
| 발표·과제 위저드 | Claude Sonnet 4.6 또는 GPT-4.1 | 품질 |
| 강의계획서 파싱 | GPT-4.1 mini + 자체 규칙 | 정확도 |
| 임베딩 | text-embedding-3-small | 비용 |

→ **무분별한 상위 모델 사용 시 무료 사용자 1인당 월 5,000원 적자 발생 가능**. 라우팅 필수.

### 3-4. 데이터 자산 보호 설계
| 자산 | 정책 |
|---|---|
| 사용자 업로드 자료 | 비공개 기본값, 학습 재사용 옵트인 분리 |
| 익명 학습 패턴 | 집계 분석만, 개인 식별 불가 |
| 강의계획서 파싱 사전 | 자체 구축, 오픈소스 공개 금지 |
| 성적·합격 추적 데이터 | 사용자 동의 + 익명 코호트 |

## 9. 100억으로 만드는 텔레로직 포인트 — 기존 기획서에 부족하던 7가지

### 9-1. 카피·메시지 전략 (기존 약함)
**문제**: "AI 학기 운영 OS"는 사업가용 정의. 학생용 카피 따로 필요.

**제안 카피 후보**:
- 메인: "**한 학기를 5분 안에 정리하는 AI**"
- 인스타: "**시험기간, 노트 다시 안 봐도 돼**"
- 친구 초대: "**같이 쓰면 둘 다 무료**"
- 시험 직전: "**3시간 후 시험, AI랑 같이**"
- 자격증: "**합격까지의 시간을 줄이는 AI**"

**원칙**: "OS"·"플랫폼" 같은 추상어 금지. **상황·고통·결과** 기반 카피.

### 9-2. Cold Start 전략 (기존 부재)
**문제**: 첫 100명·1,000명을 어떻게 모을 것인가?

**3단계 전략**:
1. **Day 1~30**: 학교 커뮤니티 침투
   - 인스타 계정 (학습 팁 콘텐츠 + 베타 모집)
   - 본인 출신 학교 학과 단톡방 (창업자가 직접)
   - 에브리타임 광고 (CPC 검토, 확인 필요)
2. **Day 31~90**: 베타 100명 확보
   - 친구 초대 코드 발급
   - 베타 사용자 1주 무료 Pro
   - "학습 후기" 인스타 콘텐츠로 확산
3. **Day 91~180**: 시험 시즌 폭발 캠페인
   - 중간고사·기말고사 1주 전 광고 집중
   - "시험 시즌 패스 3,900원" 강조
   - 시험 후 학점 향상 후기 수집·공개

### 9-3. 인플루언서·브랜딩 연계 (기존 부재)
**문제**: 사용자가 인스타·블로그 영향력 구축에 관심 있다고 했음. 본인 자산 활용해야.

**전략**:
- 창업자 본인 인스타에서 **"AI로 공부하는 법"** 시리즈 콘텐츠
- 대학생 마이크로 인플루언서 (팔로워 1만~10만) 사전 사용자
- "공부 잘하는 학생들의 AI 사용법" 콘텐츠로 권위 구축
- 본 서비스 출시 시 자연스러운 노출

→ 별도 마케팅 비용 없이 본인 브랜딩과 연계 가능. **김치형 화법: "사람들은 제품을 사는 게 아니라 사람을 산다."**

### 9-4. AI 사용 윤리·학칙 리스크 대응 (기존 약함)
**문제**: "AI로 과제 대신 작성"은 학칙 위반 가능. 우리가 위험 노출.

**해결**:
- 모든 위저드 결과물에 **"이 자료는 학습 보조용이며 학생 본인이 검토·수정해야 합니다"** 워터마크
- "리포트 초안 작성" 위저드는 **초안 직접 작성 금지**, 구조·가이드만 제공
- 학칙·교수 정책에 따른 사용자 책임 명시 (이용약관)
- "AI 활용 학습 윤리 가이드" 콘텐츠 발행 → 오히려 권위로 전환

→ Gauth가 "답 대신 풀어주는 앱"으로 포지셔닝되는 위험을 우리는 회피.

### 9-5. 한국 저작권·개인정보 컴플라이언스 (기존 충분, 강화)
**문제**: 사용자 업로드 강의자료 = 교수의 저작물. 개인정보 픽업 가능.

**필수 조치**:
- 업로드 기본 비공개, 학습 재사용 옵트인 분리
- 이용약관에 "본인이 정당하게 보유한 자료만 업로드" 명시
- 신고 즉시 24시간 내 게시중단 절차
- 개인정보 자동 마스킹 (이름·학번·연락처 패턴 검출)
- 법무 자문 1회 (50만원, 출시 전 필수)

### 9-6. 리텐션 푸시 알림 전략 (기존 부재)
**문제**: 매일 앱을 열게 하는 트리거가 모호.

**제안 푸시 시나리오**:
- 매일 오전 9시: "오늘 복습 N문제, 5분이면 끝"
- 매일 저녁 9시: "내일 마감 과제, 위저드 시작?"
- 시험 7일 전: "OO 시험 D-7, 벼락치기 위저드 추천"
- 친구 가입: "OO이 가입했어요, 같이 무료 1주!"
- 학점 입력: "이번 학기 학점 받았나요?"

**원칙**: 푸시는 하루 2회 이하. 무관한 푸시 1회만 보내도 즉시 차단됨.

### 9-7. 데이터 자산 변현 전략 (기존 약함)
**문제**: 5년 데이터 모이는데 어떻게 매출로 전환?

**제안 경로**:
1. **B2B 학습 분석 리포트**: 대학·학과별 학습 트렌드 (익명, 1억원/년 단위)
2. **자격증 학원 컨설팅**: 합격자 학습 패턴 데이터 (5,000만원/건)
3. **교과서·교재 출판사**: 어떤 단원이 어렵게 출제되는지 (수천만원/건)
4. **취업 솔루션**: 어떤 자격증·스택이 학점·취업과 상관관계 (대기업 HR)

→ 이게 **B2C 매출 60억 위에 추가로 치는 B2B 매출 50~100억**. 천억으로 가는 진짜 경로.

## 11. 결론 — 김치형 화장의 정리

> **"한 우물을 깊게, 그 우물에서 나온 물로 다른 우물을 판다."**

이 제품의 정체는 **"한국 대학생을 위한 학기 운영 OS"** 하나다. 대학생 시장에서 압도적으로 깊어진 후에야 자격증·취업·글로벌이 따라온다. 동시에 모두 가려고 하면 어디서도 1등 못 한다.

**3가지를 기억하라**:
1. **Today 대시보드의 5분이 모든 것**이다. 사용자가 매일 5분 쓰게 만들면 천억까지 간다.
2. **데이터 자산이 진짜 해자**다. 기능은 카피되지만 5년 동안 누적된 한국 대학생 학습 결과 데이터는 카피 불가.
3. **사용자가 친구를 데려오게 만들어라.** 마케팅으로 키운 회사는 마케팅으로 죽는다.
