## P0 (즉시 수정 필요 — 보안/적자/제품 정체성 직결)

### [보안] Admin Supabase 클라이언트 RLS 재검증 가드 강화 필요
- **근거**: [src/lib/supabase/admin.ts](../../src/lib/supabase/admin.ts), `src/app/api/*/route.ts` 전반
- **상황**: `getAdminSupabase()`는 RLS를 우회한다. 현재 호출처는 대부분 `getOwnerId()` 또는 `tryGetOwnerId()`로 세션 검증 후 `.eq("owner_id", ownerId)`를 추가하고 있어 **현재는 안전**. 다만 신규 라우트 추가 시 빠뜨릴 위험.
- **영향**: RLS 우회 + 다른 사용자 데이터 접근 가능
- **제안**:
  1. Admin 클라이언트 생성 함수 상단에 주석 강화: "RLS 우회 — `eq('owner_id', ownerId)` 재검증 필수"
  2. 신규 API 라우트 PR 체크리스트 추가
  3. ARCHITECTURE.md §4 RLS 정책 보완

### [비용 적자] `quiz` 위저드가 Sonnet 4.6 사용 — CLAUDE.md §1 위반
- **근거**: [src/lib/claude.ts:27](../../src/lib/claude.ts#L27) `TOOL_MODEL["quiz"] = MODELS.sonnet`, [src/lib/services/quiz.ts:96](../../src/lib/services/quiz.ts#L96)
- **상황**: CLAUDE.md §1 + PRODUCT.md §3-2는 문제 생성에 "GPT-4.1 mini 또는 Haiku 4.5" 권장. 그런데 quiz는 Sonnet 사용 중. Zod 검증 (`QuizOutput` schema)과 `validateQuiz()`로 사후 검증하고 있으므로 모델 품질이 핵심 의존이 아닐 수 있음. **무료 사용자 1인당 월 ~5,000원 적자**.
- **영향**: 적자 누적, 정책 위반
- **제안**:
  1. Haiku 4.5로 먼저 시도 → 검증 통과율 90% 이상이면 유지
  2. `exam-cram`도 함께 재평가 (현재 Sonnet — 출력 형식이 단순하면 Haiku 가능)

### [제품 정체성] presentation/report 위저드 치팅 라인 재확인
- **근거**: [PRODUCT.md §2-1 기능 6](../PRODUCT.md), CLAUDE.md §4
- **상황**: `report-checklist`는 체크리스트·구조 가이드만 제공 — **안전**. 다만 `presentation`과 `exam-cram` 프롬프트가 "본문 초안"을 생성하는지 직접 검증 못함.
- **영향**: 학교 차단, 제품 신뢰도 붕괴
- **제안**: `src/prompts/presentation.md`, `src/prompts/exam-cram.md` 재검토. "초안 작성하지 마"가 명시되어 있는지 확인.
