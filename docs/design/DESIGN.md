# DESIGN.md — arch-campus 디자인 가이드

> **사람이 만든 것처럼 보이는** 깔끔한 UI/UX. ChatGPT·Claude·Linear·Granola 톤.
> **반응형 1급 (Mobile · iPad · Desktop 동등)**. 대학생은 폰(지하철)·태블릿(카페·도서관)·노트북(스터디룸·집) 다 씀.
> Bitjaru StyleSeed의 69개 룰 중 arch-campus(학습 SaaS, 한국어, 라이트모드)에 맞는 것만 압축. 원본: [STYLESEED-FULL.md](STYLESEED-FULL.md). Linear 토큰: [styleseed-linear-tokens.css](styleseed-linear-tokens.css).

## 0. 톤 한 줄

> **"AI가 만들었구나"가 아니라 "사람이 신경 써서 만들었네"로 읽혀야 한다.**

판단 기준:
- 시니어 디자이너가 봤을 때 "장식이 너무 많네" 안 나오나?
- 색이 한 개 이상 강조되고 있지 않나?
- 그림자·라운드·여백이 과하지 않나?

## 1. 색의 절제 (★ 가장 중요)

### 1-1. 액센트는 단 하나
- **블루 단일 (`#2563eb`, `--color-accent`)**. 활성 탭, 진행률 바, 1순위 CTA, 강조 링크에만.
- 본문 텍스트·배경·일반 보더에 액센트 색 X.
- 다른 색이 필요해 보이면 그건 **회색 단계로 위계를 못 만든 것**.

### 1-2. 임팩트 색은 작게, 강하게
| 역할 | 색 | 최대 크기 | 용도 |
|---|---|---|---|
| 마감 임박·경고 | `--color-warn` `#d97706` | 도트 6px + 텍스트 11px | "D-2" 배지 |
| 완료·상승 | `--color-success` `#059669` | 도트 6px + 텍스트 13px | 성적 향상 |
| 알림 | `#ef4444` | 도트 6px (텍스트 X) | 단일 도트만 |

큰 면적에 절대 X (배지·도트만).

### 1-3. 회색 5단계 — 검정(#000)은 절대 X
| 단계 | hex | 역할 |
|---|---|---|
| Strong | `#2A2A2A` (현재 `--color-fg` `#1a1a1a`) | 가장 강한 강조 (대시보드 큰 숫자, 헤딩) |
| Primary | `#3C3C3C` | 본문 메인 텍스트 |
| Secondary | `#6A6A6A` (현재 `--color-fg-muted` `#6b7280`) | 캡션·라벨 |
| Tertiary | `#7A7A7A` (현재 `--color-fg-subtle` `#9ca3af`) | 부제, 날짜, 설명 |
| Disabled | `#9B9B9B` | 비활성, placeholder |

**우리 globals.css 토큰은 이 단계와 거의 일치**. 새 색 추가 X. 위계가 부족하면 단계를 더 쓰지 새 색을 만들지 X.

### 1-4. 배경 위계
| 역할 | 색 |
|---|---|
| 페이지 | `#FAFAFA` 또는 `#FFFFFF` (현재: 흰색 메인 + 사이드바 `--color-surface`) |
| 카드 | `#FFFFFF` |
| 비활성 trail | `--color-border` |
| 액센트 tint (선택 행) | `--color-accent-soft` `#eff6ff` |

## 2. 텍스트 위계 5단계

| 레벨 | 크기 | weight | 용도 |
|---|---|---|---|
| Display | 28px | 600 | 페이지 제목 ("좋은 오후예요…") |
| H1 | 20px | 600 | 카드 메인 텍스트 (5분 카드 본문) |
| H2 | 14~15px | 500 | 리스트 아이템 제목 |
| Body | 13~13.5px | 400 | 본문 |
| Caption | 12~12.5px | 400 | 부제, 날짜, 메타 |
| Micro | 11px | 500 | 배지, 섹션 라벨 (대문자) |

**규칙**:
- 한 화면에 6개 이상 크기 X
- 차이는 **2~4px**씩만. 12 → 14 → 16 → 20 → 28
- 한국어 + Pretendard라면 letter-spacing은 -0.01em ~ 0 (영문보다 살짝 좁게)

---

## 3. 숫자 표기 (학점·점수·진행률)

**큰 숫자 + 작은 단위, 2:1 비율**, 단위는 `ml-0.5` 정도 띄우기.
- 히어로 메트릭: 48px / 24px (예: `3.8M`)
- KPI 카드: 36px / 18px (예: `4.2/4.5`)
- 진행률 텍스트: 13px / 13px 같은 크기 (`5/12`)

폰트: `tabular-nums` 클래스 필수 (숫자 자리 정렬).

---

## 4. 그림자 — 거의 안 보이게

- 카드: `box-shadow: 0 1px 2px rgba(0,0,0,0.04);` 정도. 호버 시에만 살짝 더.
- **opacity 4~8% 이상이면 과함**.
- "그림자 보인다" 싶으면 이미 너무 많이 줬다.
- 보더(`1px var(--color-border)`)가 그림자보다 우선. **분리는 그림자가 아니라 배경 차이로**.

---

## 5. 카드 구조

```
┌──────────────────────────────┐
│ Header (메타·라벨)              │  - 12px secondary
│                                │
│ Content (메인 정보)            │  - 14~20px primary
│                                │
│ Footer (액션·진행률)           │  - 12~13px tertiary
└──────────────────────────────┘
```

**규칙**:
- 한 카드에 정보 4개 이상 X. 넘치면 두 카드로 분할.
- 제목·메타 사이 여백 2~6px. 메타·본문 사이 8~12px. 본문·액션 사이 16~20px.
- 카드 라운드: `rounded-xl` (12px) 일관. 큰 카드(메인 CTA)만 `rounded-2xl` (16px).

---

## 6. 리스트 아이템

```
[● 색도트] [메타: 과목] [배지: D-2]
          [본문 한 줄: 자료구조 과제 — 이진 탐색 트리]
                                              [hover: 액션 →]
```

- 도트: 과목 색 (sidebar 색과 일치)
- 배지: 위 §1-2 임팩트 색 규칙대로
- 호버 시에만 우측 액션 노출 (Linear·Granola 스타일)
- 행 높이: 56~64px (`py-4`)

## 7. 버튼

| 변형 | 용도 | 스타일 |
|---|---|---|
| Primary (어두운) | 카드 메인 CTA ("시작") | `bg-fg text-white` |
| Accent | 결제·전역 CTA | `bg-accent text-white` |
| Ghost | 행 호버 액션 | `text-fg-muted` (호버 시 노출) |
| Outline | 보조 액션 | `border bg-white` |

**규칙**:
- Primary는 한 화면에 최대 2개.
- 같은 위계의 버튼이 3개 이상 보이면 위계를 다시 짜라.
- 모서리: 8px (`rounded-lg`) 일관.

---

## 8. 여백 리듬 (★ 단조로움 깨기)

같은 섹션 타입을 절대 두 번 연달아 X. 우리 Today 페이지는 이미 잘 되어있음:

```
[히어로 카드: 큰 그라데이션] ← 강조
   ↓
[리스트 카드 3개]            ← 평면
   ↓
[리스트 카드 1개]            ← 평면
   ↓
[리스트 카드 3개]            ← 평면
```

3번째·4번째가 같은 형태라 단조롭다 — **5번째 섹션에는 다른 패턴**(예: 가로 스크롤 카드, 큰 일러스트, 통계 위젯)을 넣어 리듬을 깬다.

페이지당 섹션 4~7개. 4 미만이면 빈 페이지, 7 초과면 산만함.

## 9. 마이크로카피 톤 (한국어)

**원칙**: 친근한 반말 X, 정중한 평어 O. ChatGPT·Toss 스타일.

| 좋음 | 나쁨 |
|---|---|
| "오늘 5분이면 끝나는 일부터" | "5분 안에 빠르게 끝내세요!" |
| "지금 시작" | "지금 바로 시작하기" |
| "확인이 필요해요" | "주의: 확인 필요!" |
| "복습 12문제" | "12개의 복습 문제가 있습니다" |

- 능동태, 간결, 긍정형
- 느낌표 페이지당 0~1개
- "여러분"·"우리" 자제. "OO님"으로 직접 호명

## 10. 절대 금지 (StyleSeed의 30+ Don'ts 중 핵심)

- ❌ `#000` 검정 (가장 진한 회색은 `#2A2A2A`)
- ❌ 진한 그림자 (`0 4px 8px rgba(0,0,0,0.1)` 같은)
- ❌ 액센트 색 두 개 이상
- ❌ 본문에 액센트 색
- ❌ 큰 면적에 경고/임팩트 색
- ❌ 한 카드에 4개 초과 정보
- ❌ 페이지에 같은 카드 형태 4개 연달아
- ❌ 무지개 색 (5색 이상 차트)
- ❌ 라운드 16px 초과 (이미지·아바타 제외)
- ❌ AI 생성 일러스트·이모지 본문 사용 (섹션 헤더 작은 아이콘만 OK)
- ❌ 그라데이션 남용 (히어로 카드 1곳만)
- ❌ 폰트 5종류 이상

## 13. 반응형 전략 (★ 1급 시민)

대학생은 **모바일·태블릿·데스크톱 다 씀**. 단일 코드베이스, 화면 크기로만 분기.

### 13-1. 브레이크포인트 (Tailwind 기본 + 우리 정책)

| 토큰 | 너비 | 대표 디바이스 | UX |
|---|---|---|---|
| (none) | < 640px | iPhone (375~430), 갤럭시 | **모바일 풀** |
| `sm:` | ≥ 640px | 작은 태블릿 가로, 폴더블 | 모바일 + 여백 살짝 |
| `md:` | ≥ 768px | iPad 세로 (820 포함) | **태블릿** — 사이드바 등장 |
| `lg:` | ≥ 1024px | iPad 가로, 작은 노트북 | 데스크톱 진입 |
| `xl:` | ≥ 1280px | 일반 노트북·모니터 | **데스크톱 풀** |

### 13-2. 네비게이션 — 좌측 사이드바 ↔ 하단 탭바

- **`md` 이상**: 좌측 사이드바 240px (현재 구현)
- **`md` 미만 (모바일)**: 사이드바 숨김 + **하단 탭바**(56~64px, 4탭) + 상단에 mini 헤더(로고 + 햄버거 또는 사용자)
- 하단 탭바는 iOS Safe Area 고려 (`pb-[env(safe-area-inset-bottom)]`)
- 사이드바의 "이번 학기 과목" 리스트는 모바일에선 햄버거 메뉴 또는 Today 페이지 상단 chip 그룹으로 이동

### 13-3. 컨테이너 너비

| 화면 | 메인 컬럼 max-width | padding |
|---|---|---|
| 모바일 | 100% | `px-4` (16px) |
| 태블릿 | `max-w-2xl` (672px) | `px-6` |
| 데스크톱 | `max-w-3xl` (768px, 현재) | `px-10` |

대시보드는 한 컬럼 유지. 정보 밀도가 데스크톱에서도 한 컬럼이 맞음 (ChatGPT·Claude 패턴).

### 13-4. 카드·리스트 모바일 변환

- 데스크톱의 호버 노출 액션 → 모바일에선 **항상 노출** 또는 행 자체가 탭(전체 행 터치 영역)
- D-day 배지·도트는 그대로 (작아도 가독)
- 카드 내부 padding: 데스크톱 `p-5~p-6` → 모바일 `p-4`
- 5분 메인 카드의 가로 정렬(`flex items-end justify-between`)은 모바일에서 `flex-col gap-4`로 분기

### 13-5. 폰트 크기

대학생은 작은 폰을 가까이 본다 — **모바일에서 키우지 말고**, 위계 그대로 유지하되 한 단계만 살짝.

| 레벨 | 모바일 | 데스크톱 |
|---|---|---|
| Display | 24px | 28px |
| H1 | 18px | 20px |
| H2 | 14px | 14~15px |
| Body | 13.5px | 13.5px (동일) |
| Caption | 12px | 12px (동일) |

### 13-6. 터치 타깃

- **모든 인터랙티브 요소 44×44px 이상** (Apple HIG 기준)
- 체크박스·도트·아이콘 자체는 작아도, 터치 영역은 패딩으로 확보
- 인접 버튼 사이 8px 이상 간격

### 13-7. iPad 특이사항

- iPad 세로(820px)는 `md:` 진입 → 사이드바 등장. 하지만 **풀 240px이 너무 차지함**.
  - 옵션 A: `md:`는 사이드바 압축 (아이콘만 64px) + `lg:`부터 풀 240px
  - 옵션 B: `md:`도 풀 사이드바 유지하되 메인 컬럼 좁힘
  - **현재 권장: 옵션 A** (정보 밀도 우선)
- iPad의 외부 키보드·트랙패드 사용도 고려 — 호버 상태 정상 동작 확인

### 13-8. 검증 워크플로 — 새 페이지 끝낼 때마다

```bash
# 1. 데스크톱 (이미 보고 있음)
# 2. 태블릿: Chrome DevTools → iPad Air (820×1180)
# 3. 모바일: Chrome DevTools → iPhone 14 (390×844)
```

또는 Playwright MCP가 설치돼 있으면:
- "iPhone 14, iPad Air, Desktop 1280에서 /dashboard/today 스크린샷 찍어줘" 한 줄로 끝
- 호버·탭·키보드 인터랙션도 자동 검증
